Overview

This program is a terminal-style automation hub with password protection, an encrypted Anime Vault, Chrome launchers, a key macro tool, a macro recorder/player, a file runner, and more.

Boot Sequence Demo

When animations are enabled in Settings, you'll see this on each launch:

> SYSTEM BOOT
Boot animation — plays on every launch (if animations enabled)

What gets created on first run

If the %AppData%\MainGUI folder or any required file is missing, the program creates it automatically with safe defaults.

  • Default password set to 1234 (stored encrypted).
  • configs.cfg created with lock timer = 60s, max fails = 3, profile = 7.
  • oooh.bat helper created for logging wrong password attempts.
  • authorize.cfg and sites.cfg created empty.
  • Tray icon set from icon.png in the script directory (if it exists).

Getting Started

First run

  1. Double-click the program EXE to run.
  2. The boot animation plays (dots loading: Loading assets… Unpacking modules… Injecting core…).
  3. The ACCESS PORTAL password dialog appears. Enter 1234 on first run.
  4. After a successful login, the boot sequence plays again (Authorizing session… Activating interface…) then the main GUI opens.

Login details

  • Password is case-insensitive — "HELLO" and "hello" are the same.
  • Check Show Password in the login dialog to reveal what you typed.
  • Failed attempts are counted; too many trigger a timed lockdown.

Open / Exit / Restart

  • The main GUI is titled ENCRYPTED TERMINAL. Click any button to access a feature.
  • Click Exit Terminal to close the program.
  • To reload after editing config files, close and reopen the EXE.

Password & Security

Access is protected by an encrypted password with a fail counter and automatic timed lockout system.

How lockout works

  • Each wrong password increments a counter in systems32.sys.
  • When the counter reaches a multiple of Max Failed Attempts, a lockdown is triggered.
  • Lockdown duration grows: 1× = base duration, 2× = 2× base, and so on.
  • At 1000 failures, the lock is set for 100 years (permanent lockout).
  • Lock expiry is stored in winlogxs.dll as a UTC timestamp.
↑ Visualised: fail counter progress toward lockout (resets on success)

Wrong password logging

  • If Enable Logs is on in Settings, each wrong attempt is written to knernals32.txt via oooh.bat.
  • Log entries include the UTC timestamp and whatever was typed.
  • The batch file is auto-created if missing.

How encryption works

Passwords and Anime Vault titles use an affine cipher — each letter is transformed by the formula (3x + 2) mod 26. The reverse (decryption) finds x for each encrypted character.

plaintext a (x=1) → (3×1+2) mod 26 = 5 → encrypted e

You don't need to understand this to use the program. Just set your password via Settings.

Change password

  1. Open Settings → Change Access Key.
  2. Enter the old password and the new password twice to confirm.
  3. The new password is immediately saved encrypted to logoffs.dll.

If you lock yourself out

Delete %AppData%\MainGUI\winlogxs.dll to clear the lock. Your password is unchanged. To reset the fail counter, delete systems32.sys.

Daily Auto-Backup

When line 8 is true, a dated copy of your %AppData%\MainGUI folder is saved to %AppData%\backup\Backup_YYYY-MM-DD_… once every 24 hours. The last backup time is tracked in backup_time.txt (in the script directory).

Manage site lists

Anime Vault

Store and retrieve encrypted anime/media titles. The vault is protected by the same affine cipher as the password. Never edit logons.dll manually.

Add or edit titles

  1. From Main Menu, click Access (Retrieve / Submit).
  2. Select Edit Anime List.
  3. Enter one title per line.
  4. Click Save & Encrypt. The file is re-written encrypted.

Search a stored title

  1. Click Access → Search Anime Entry.
  2. Select a radio button next to the title you want.
  3. Click Search Selected Title — it opens a Google search for that title in a Chrome guest session.

Access control

The Anime Vault can be individually locked/unlocked. Use the ⚿ lock button in the main menu to unlock all sections at once, or access it directly (the vault status shows Locked / Unlocked on the main screen).

File Runner (Manage File Runner)

Save shortcuts to files you use often and launch them in one click. File paths are stored in %AppData%\MainGUI\files.txt.

How to use

  1. Click Manage File Runner from Main Menu.
  2. Click Open File to browse and save a file path. Duplicates are detected and skipped.
  3. Click Saved Files to see all saved entries. Click any filename button to launch it.
  4. Click Delete Entries to remove a saved entry from the list.

Notes

  • Duplicate detection is case-insensitive.
  • If a file is missing from disk, you'll see an error when trying to run it.
  • File paths are stored in files.txt, one per line.

Troubleshooting

GUI doesn't appear
  • Right-click the EXE → Run as administrator.
  • Check Windows Defender / antivirus: it may quarantine the script runner.
  • Make sure AutoHotkey v1 is installed on the machine.
Sites won't open in Chrome
  • Make sure Chrome is installed and chrome.exe is on the system PATH.
  • All URLs must include http:// or https://.
  • Check that the Profile Number in Settings matches a real Chrome profile folder.
Password accepted but nothing happens
  • Check if the boot animation is stuck — it may be disabled if Disable Animations is true.
  • Restart the program. The main GUI (ENCRYPTED TERMINAL) should appear.
Locked out and timer hasn't expired
  • Navigate to %AppData%\MainGUI and delete winlogxs.dll.
  • Also delete systems32.sys to reset the fail counter.
  • Restart the program — the lockdown is now cleared.
Anime Vault shows garbled text

Do not manually edit logons.dll. If it is corrupted, delete it. The program will create a fresh empty file. Re-add your titles via the Edit Anime List option.

FAQ

Q: Can I edit config files manually?

A: Yes for sites.cfg, authorize.cfg, and click_config.cfg. Always use the program for logoffs.dll (password) and logons.dll (Anime Vault) — these are encrypted.

Q: I forgot my password. What do I do?

A: Delete %AppData%\MainGUI\logoffs.dll. On the next run, the program will create a new file with the default password 1234. You will lose any previously set password.

Q: Why are files named .dll or .sys?

A: They are plain text files. The extensions are used purely for disguise. They will not affect Windows and are not real system libraries or drivers.

Q: What is the backup feature?

A: When line 8 of configs.cfg is true, the program backs up %AppData%\MainGUI to %AppData%\backup once per day. Backups are dated folders. You can restore by copying files back manually.

Q: Can I run multiple Chrome profiles at once?

A: The custom site launcher uses one profile (set in Settings). The guest launcher always opens a guest session. Both can run simultaneously for separation between personal and browsing sessions.

Q: What does the ⚿ lock button do exactly?

A: It asks for your password and unlocks all three areas (Vault, Chrome, Settings) in one go — useful so you don't have to authenticate three times separately. Clicking it again when unlocked re-locks everything.
